<task type="auto">
  <name>Task 1: Create validate-data.rb with schema validators</name>
  <files>scripts/validate-data.rb</files>
  <action>
    Create scripts/validate-data.rb as a Ruby module invoked by Rake. Use Psych safe_load (never YAML.load with arbitrary types) per research security guidance.

    Implement a DataValidator class with:
    - exit-code semantics: 0 pass, 1 fail (print human-readable errors to STDERR)
    - validate_file_exists helper for expected _data/ paths
    - validate_navigation(data): array of hashes with required keys title, url, order; exactly 4 items; all urls lowercase ending in /; must include /about/, /work/, /career/, /blog/ per D-18 and D-19
    - validate_social(data): required keys name, email, github, linkedin, twitter, gravatar_hash; github/linkedin/twitter must have url subkeys
    - validate_projects_featured(data): array length 3-5 per D-01; each entry requires id, title, summary, tech_stack (non-empty array), github_url, demo_url (nullable), demo_status (live|code_only|broken); reject any github_url containing UW-CSS per D-03
    - validate_projects_archive(data): array; same required fields as featured minus count limit
    - validate_career(data): education array with institution+degree only per D-09; roles array max 3 per D-07; each role requires employer, title, start_date, end_date, bullets (min 1); if status != pending_linkedin then each bullet must match quantified metric pattern (contains digit or % per D-08)
    - validate_blog_inventory(data): meta.total_posts equals posts.length; posts array non-empty check stub (full 51-post coverage enforced in Plan 04)

    Main runner: accept --check flag values (all, navigation, social, projects, career, inventory) default all; skip missing files with clear SKIP message (data files created in later plans).
  </action>
  <verify>
    <automated>ruby -c scripts/validate-data.rb && ruby scripts/validate-data.rb --check navigation 2>&1 | head -5</automated>
  </verify>
  <done>validate-data.rb parses cleanly and runs without Ruby syntax errors; missing _data/ files produce SKIP not crash</done>
</task>
